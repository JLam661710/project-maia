# 产品需求文档 (PRD)：AI驱动全自动化选题决策系统

## 核心价值主张
为独立内容创作者提供全自动化的AI驱动选题决策系统，基于小红书/B站/微信公众号实时数据生成带涨粉潜力评估的每日优先级选题清单，彻底摆脱凭感觉选题的依赖，快速打造可变现内容账号。

## 场景故事化 (3-Min Story)
凌晨1点，小红书和B站双平台创作者阿凯坐在电脑前，刷新了第17次热点榜单。过去一周，他凭直觉选的3条视频只有不到1000播放，涨粉数停留在2300，距离变现门槛的1万粉遥遥无期。他试过手动分析竞品，但花了5小时整理的数据还是没找到规律；用爬虫工具爬热点时又被平台限制，导致选题耽误了整整一天。看着银行卡里的余额，他焦虑得睡不着：“凭我一个人，怎么可能跟得上平台变化？要是有个工具能直接告诉我今天该做什么内容就好了……”

## 用户故事
1. As a 独立内容创作者, I want 每天收到带优先级排序的选题决策清单, so that 我无需花费3小时以上手动找热点、分析数据，能直接聚焦内容生产。
2. As an AI-native内容运营者, I want 系统自动切换爬虫工具绕过平台数据限制, so that 不会因为爬取失败导致选题延误，保证每日推送的稳定性。
3. As a 追热点的内容创作者, I want 每个选题都附带涨粉潜力等级、适合的内容格式和核心受众标签, so that 我能优先投入高价值选题，快速提升涨粉速度。
4. As a solo内容运营者, I want 系统在部分数据爬取失败时自动跳过并继续生成报告, so that 不会因局部数据缺失影响整体选题决策。

## 本质需求
用户渴望**无需依赖个人创作能力、经验或主观判断，通过AI与数据的全自动协作，快速打造具备稳定变现能力的内容账号**。成功标准为：涨粉数量达到平台变现门槛（如1万粉），涨粉速度较手动运营提升50%以上。

## 功能清单
### P0 (Must-have)
1. 多平台数据爬虫引擎：支持小红书、B站、微信公众号数据爬取，内置自动切换机制（扣子工作流爬虫+备用工具）绕过平台限制
2. 实时热点趋势跟踪：基于爬取数据识别各平台当日/当周热点话题、关键词热度变化
3. 选题决策引擎：生成带以下属性的选题建议：
   - 涨粉潜力等级（高/中/低）
   - 适配内容格式（短视频/图文/长视频/公众号文章）
   - 核心受众标签（如“Z世代美妆爱好者”“职场新人”）
4. 每日定时推送：按用户预设时间（如每日早8点）将优先级排序的选题清单推送到指定渠道
5. 异常容错机制：数据爬取失败时自动跳过该数据源，继续生成完整选题报告

### P1 (Nice-to-have)
1. 竞品成长轨迹分析：追踪同领域Top账号的内容选题、发布频率与涨粉关联数据
2. 评论情感分析：分析热点话题下用户评论的情感倾向，辅助判断选题的受众接受度
3. 多平台格式适配建议：针对不同平台特性，为同一选题提供差异化内容格式优化建议
4. 变现潜力评估：结合选题热度与平台变现规则，预估内容的广告、带货等变现潜力

## 产品化判断
**当前阶段为自用工具，具备可复用方案属性，未来可产品化为SaaS服务**。
- 依据：用户当前为solo运营，核心需求是解决自身选题效率问题；但选题决策是所有内容创作者的通用痛点，系统的AI驱动架构、多平台适配能力具备高度可复用性，验证效果后可快速包装为面向中小创作者的付费工具。

## 验收标准
1. 核心指标：涨粉速度较手动运营提升50%以上
2. 效率指标：选题决策时间从3小时/天降至10分钟/天以内
3. 稳定性指标：爬虫任务成功率≥90%，每日选题推送准时率100%
4. 质量指标：用户选择的Top3选题播放量/阅读量较历史平均提升30%以上
